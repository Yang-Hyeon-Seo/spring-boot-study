# 스프링 이니셜라이저
- 모든 스프링 부트는 스프링 이니셜라이저 도구를 사용해 생성하는 것부터 시작함
- [스프링이니셜라이저](https://start.spring.io/)
- 스프링 이니셜라이저 화면의 좌측에는 프로젝트를 설정하는 영역이 있고, 우측에는 의존성 라이브러리를 추가하는 영역이 있음
## 프로젝트 설정
- project
  - 빌드 시스템 설정
  - 메이븐 vs 그래들
  - 그래들 빌드 시스템
    - 빌드 설정을 위한 언어로 그루비와 코틀린을 선택할 수 있음
    - 스프링 애플리케이션에서는 일반적으로 그루비를 사용함
- Language
  - 애플리케이션을 개발할 때 사용할 프로그래밍 언어
  - 자바, 코틀린, 그루비 모두 자바 VM 기반 언어
  - 스프링 애플리케이션에서는 일반적으로 자바 언어를 많이 사용함
  - 스프링 이니셜라이저로 프로젝트를 생성하면 스프링 부트 메인 클래스가 함께 생성되며, 이때 사용되는 언어이기도 함
- Spring Boot
  - 사용할 스프링 부트의 버전 선택
  - 가급적 버전명 뒤에 SNAPSHOT, M1, M2와 같은 **꼬리표가 붙지 않은** 정식 버전을 선택하는 것이 좋음
    - SNAPSHOT : 개발 중인 버전. 안정성이 낮고 변경될 가능성이 높아 가능하면 사용하지 X
    - M1, M2, ... (마일스톤): 특정 목표를 구현하고 테스트한 버전. 어느 정도의 안정성은 확보했으나 여전히 변경될 가능성이 높음
    - RC1, RC2, ... : 정식 버전을 앞둔 후보 버전. 마일스톤보다 안정적이고 심각한 버그만 수정되며 기능의 추가는 없음
- Project Metadata
  - Group
    - 프로젝트가 속한 조직이나 도메인
    - 일반적으로 도메인 이르을 거꾸로 사용함
    - ex: org.hanbit
  - Artifact
    - 특정 프로젝트의 고유한 이름
    - 동일한 Group 내에서 프로젝트를 구분하는 역할 수행
  - Name
    - 애플리케이션 이름
    - 일반적으로 Artifact와 동일한 이름 사용
    - 필요에 따라 다르게 사용할 수 있음
    - Name으로 지정된 이름은 프로젝트에 기본적으로 생성되는 메인 클래스 이름으로 사용됨
    - *마이크로 서비스*를 구축할 때는 *유레카 서비스* 등에서 애플리케이션을 구분하는 아이디로 사용되기도 함
  - Description
    - 프로젝트를 설명하는 부분
    - 메이븐 빌드 시스템을 선택한 경우 빌드 설정 파일에 부가적으로 입력되기도 하지만, 그래들 빌드 시스템에서는 실제로 저장되거나 사용되지 X
  - Package name
    - 일반적으로 Group 이름과 Artifact 이름의 조합으로 사용됨
    - 자바 프로젝트 기본 패키지 이름으로 사용됨
  - Packaging
    - 스프링부트 프로젝트에서는 Jar(Java Archive) 포맷과 War(Web Archive) 포맷 두가지 지원
    - 스프링 웹 애플리케이션의 경우 Jar 포맷으로 패키지를 생성하면 톰캣과 같은 웹 애플리케이션 서버를 내장해 편리하게 웹 애플리케이션을 실행할 수 있음
    - 만약 톰캣과 같은 웹 애플리케이션 서버에 배포해야 한다면 War 포맷 선택
  - Java
    - 프로젝트에 사용할 자바 버전 선택
    - 가급적 LTS를 선택하느 것이 좋음
- 우측영역은 애플리케이션에서 사용할 의존성 라이브러리를 추가할 수 있음
  - Spring Web : 웹 애플리케이션을 만들기 위해 필요한 의존성 라이브러리

# 프로젝트 시작 전 알아두어야 할 사전 지식
- Getter, Setter → 비즈니스 로직이 아니면서도 꼭 필요해서 반복적으로 사용하는 코드 = **보일러 플레이트**
- `롬복` : 이러한 보일러플레이트 코드를 더 스마트하게 자동 생성해주는 도구
